The Task

2021-01-30 :: rails, ruby, tutorial

The application I planning to create is based on some code assignment task. The requirements for the assignment are as follows:

The service is a classified ads website. Users can create ads and submit them for moderation. Admins can view users and ads awaiting moderation. The admin can confirm the publication or reject it with a comment. Actions with the ad are logged. All published ads are displayed on the main page and can be filtered and sorted.

The main entities are:

  • User
  • Ad
  • Ad category

Ads

The ad has a title, description, several images, price.
The ad can only be in one of the categories.
The ad can be one of five types: “sale”, “purchase”, “exchange”, “service”, “rental”.
The ad can be in one of the states: “new”, “pending publication”, “rejected”, “published”, “archive”.
Every 10 minutes all ads that are more than 1 day in the “published” state should be transferred from “published” to “archive” state.
All actions (who, when, from which to which state, comment, if any) with ads must be logged.

Guests

Guests have access to the main page, can not create ads.

Users

The user can register on the site.
The user must have an avatar image.
The user can create ads. Immediately after creation, the ad is in the state “new”.
The user has an account page in which he can see his ads and their status.
When an ad is edited it automatically goes to “new” status.
User can switch ads from “new” and “archive” status to “pending publication” state, as well as from any state to “archive” state.

Administrator

The administrator can create and edit ad categories.
The administrator can view a list of all users and their ads in “pending”, “rejected”, “published” status.
The administrator has a control panel that displays all ads from users in “pending publication” status.
The administrator can move user ads from “pending publication” status to “published” or “rejected”.
When an ad is rejected by an administrator, he must specify a comment, which will be visible in the ad action log.

Main page

The main page is a list of ads from different users in “published” state. Ads can be sorted by publication date and price.
Ads can be filtered by category and status simultaneously or separately.
It should be possible to find the ad by content.
