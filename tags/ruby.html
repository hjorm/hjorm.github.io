Rack microservice

2024-02-04 :: ruby, rack, basics

I recently came across this coding assignment of creating a microservice in Ruby. I found it interesting and a bit challenging.

Assignment text under spoiler
  • You need to create an endpoint for a POST request that will accept requests with data in JSON format.
  • Incoming request must contain information about the user. The data structure can be defined by you, but it must follow the JSON format.
  • Microservice should validate the received data and save it to a database. You can choose any database you prefer to work with.
  • If data is successfully saved, then microservice should return a status of 201 Created and a URL to the saved data.
  • You should create a GET request endpoint that will return a list of stored records
  • You can add authentication and authorization to the microservice (optional)
  • Pay special attention to the minimalism of your code. The fewer lines, classes, files and gems, the better.

The main requirement - minimal use of libraries, gems and generally minimal code - got me interested. It is really easy to create the necessary functionality using Rails, Hanami, Sinatra etc., but I’ve never done anything like this with Rack.
